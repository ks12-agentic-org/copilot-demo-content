## Demo 1 — Rough Notes → Professional Report (0:48–0:50)

```demo
File: Engineering_Meeting_Notes_Raw.docx — absichtlich chaotische Notizen (Abkürzungen, kein Format, Deutsch/Englisch gemischt).

Schritt: Word öffnen → Datei hochladen → Copilot Sidebar → Prompt.

💬 Say: "So kommen Meeting-Notes rein. So kommen sie raus."
```

### Datei öffnen + Copilot Sidebar → Prompt eingeben

```prompt
[Upload: Engineering_Meeting_Notes_Raw.docx]

Turn these meeting notes into a professional project status report for management. Structure it with:
- Executive Summary (3 bullet points, max 60 words)
- Key Decisions Made (bullet list)
- Open Action Items (table: Action | Owner | Due Date | Status)
- Risks & Escalation Items

Professional language, ready to share with the VP Engineering. No abbreviations.
```

### Follow-up: Andere Audience, gleicher Inhalt

```demo
Gleiche Datei — anderes Publikum. Zeigen wie Copilot denselben Content transformiert.
```

```prompt
Now rewrite this as a 1-page board summary. The board has no engineering background. Focus on:
- Revenue at risk (financial impact of the quality issue)
- What decisions the board needs to make today
- What success looks like in 30 days

Cut all technical detail. Maximum 250 words. Business-executive language only.
```

### Copilot in Word — Reference document → generate new content

```prompt
[Upload: Generic_Product_Brief.docx]

Based on this product brief, write a 2-paragraph customer-facing summary I can use as an introduction in a sales proposal. Tone: confident, benefits-focused, no internal jargon. Highlight: 800V support, weight reduction, ISO 26262 certification, and Q4 2027 availability.
```

```prompt
Now draft a 3-question FAQ about this product — the questions a procurement manager would ask. Include clear, short answers based on the document content.
```
